## Overview

FoodIndex is a full-stack MERN marketplace where restaurants and cafes list surplus food, and nearby buyers purchase it at prices that decrease automatically as pickup time approaches. No food gets thrown out, buyers save money, and sellers recover costs.

The standout feature is the **live pricing engine**: prices are never stored in the database. Every time a product is fetched, a 4-factor algorithm recomputes the exact price for that moment — factoring in time remaining, stock pressure, demand signals, and urgency. No cron jobs. No stale prices.

### Key Features

- **Live Pricing Engine**: price computed fresh on every product fetch — time decay, stock pressure, demand, and urgency
- **Price Chart**: Recharts area chart showing real price history from purchase events + current live price
- **Location-based Marketplace**: geospatial `$near` query to surface listings within 5km
- **Demand Signals**: view counts and watcher count feed into the pricing algorithm in real time
- **Seller Dashboard**: full inventory management with inline editing, image upload, and order status control
- **Buyer Dashboard**: order history with savings tracking
- **Dark Mode**: CSS variable–based theming with system preference detection
- **Cart Staleness Detection**: cart re-fetches live prices on load and flags expired or price-changed items
- **JWT Authentication**: protected routes for buyers and sellers with role-based access

## Architecture

```
client/                         server/
├── src/                        ├── src/
│   ├── context/                │   ├── features/
│   │   ├── AuthContext.jsx     │   │   ├── auth/
│   │   ├── CartContext.jsx     │   │   ├── products/
│   │   └── ThemeContext.jsx    │   │   ├── orders/
│   ├── pages/                  │   │   └── uploads/
│   ├── components/             │   ├── models/
│   └── lib/                    │   ├── utils/
│       ├── api.js              │   │   └── pricingAlgorithm.js
│       └── queryClient.js      │   └── server.js
└── vite.config.js              └── seeder.js
```

**Request flow**: React (TanStack Query) → Axios (`/api` proxied to `:5000`) → Express → MongoDB Atlas. Price is injected by `pricingAlgorithm.js` on every product response — never read from, only written to on purchase events.

## Quick Start

### Prerequisites

- Node.js 18+
- MongoDB Atlas account (free tier works) — or local MongoDB

### 1. Clone the repo

```bash
git clone https://github.com/Gin5115/FoodIndex.git
cd FoodIndex
```

### 2. Set up the server

```bash
cd server
npm install
```

Create `src/.env`:

```env
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/foodindex
JWT_SECRET=your_secret_here
PORT=5000
```

### 3. Seed the database

```bash
npm run seed
```

This creates 3 seller accounts, 2 buyer accounts, and 9 products with realistic price history generated by the live algorithm.

```
Sellers: arjun@thecornerbakery.com / priya@saffronkitchen.com / ravi@brewmornings.com
Buyers:  kavya@email.com / sam@email.com
All passwords: password123
```

### 4. Start the server

```bash
npm run dev
```

### 5. Set up the client

```bash
cd ../client
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

| `JWT_SECRET` | Yes | Secret used to sign JWT tokens |

The Vite dev server proxies `/api` to `http://localhost:5000` by default. Change the target if your server runs on a different port.

## How the Pricing Engine Works

Every `GET /api/products/:id` call runs this algorithm — no stored price is ever read:

```
lifeFraction  = elapsed / totalWindow          (0 at listing → 1 at pickup)

timeDecay     = lifeFraction × 0.40            cheaper as time runs out
stockPressure = (stock / initialStock) × 0.30  overstocked = deeper discount
demandFactor  = (1 − demandScore) × 0.20       high demand = less discount
urgency       = ((60 − minutesLeft) / 60) × 0.10  final hour: extra cut

totalDiscount = min(sum of factors, 0.70)      hard cap at 70% off
currentPrice  = max(original × (1 − discount), original × 0.30)
```

**Key behaviors:**
- A product sitting at full stock with no demand is priced at \~50% off from listing
- Selling units reduces `stockPressure`, which can cause the price to tick *up* even as time passes — the algorithm rewards popular items
- The urgency factor only activates in the final 60 minutes, adding up to 10% more discount
- Price history is written **only on purchase events** — the chart shows meaningful price points, not polling noise
